# 01 产品概述

## 产品定位

**幸运大抽奖** 是一款轻量级的**在线抽奖/转盘活动**产品：活动创建者配置转盘与奖品、生成分享链接，参与者打开链接即可参与抽奖，结果可被记录与查看。

- **一句话**：创建抽奖活动 → 分享链接 → 他人参与 → 查看中奖记录。
- **典型场景**：生日/节日礼物抽奖、团建/年会、线下活动互动、社群福利发放。

## 目标用户

| 角色 | 描述 | 核心诉求 |
|------|------|----------|
| **创建者** | 发起活动的人（如寿星、活动组织者） | 快速创建、易配置、可分享、能看记录 |
| **参与者** | 通过链接打开并抽奖的人 | 无需注册、流程短、结果清晰、可兑奖 |

两者可能重叠（创建者也可参与自己的活动），但产品需同时优化「管理侧」与「参与侧」体验。

## 核心价值

1. **零门槛参与**：参与者无需登录，点链接即玩，降低流失。
2. **创建简单**：表单与配置收敛、默认值可用，支持「创建即分享」。
3. **氛围感**：公开抽奖页可带开场文案、头像、转盘与中奖弹窗，适合节日/生日场景。
4. **可追溯**：中奖记录可查、可搜索，便于兑奖与统计。

## 功能边界（MVP）

- 创建者：注册/登录 → 创建活动 → 编辑（标题、分享 slug、转盘奖品、开场文案等）→ 查看某活动的抽奖记录（列表、排序、搜索）。
- 参与者：打开 `/l/[slug]` → 观看开场（可选）→ 转盘抽奖 → 查看中奖结果 → 关闭；中奖写入记录。
- 不覆盖：支付、实物发货、多轮次/规则引擎、企业 SSO 等；后续可在此基础上扩展。

## 成功指标（可量化）

- 创建者：从注册到首次分享链接的完成率、单用户活动数。
- 参与者：从打开链接到完成一次抽奖的转化、单活动参与人次。
- 体验：移动端占比高，需保证小屏可用性与加载速度。
