# 02 用户旅程与关键流程

## 角色与入口

- **创建者**：从落地页或直接访问后台入口 → 登录/注册 → 进入「我的活动」或「新建活动」。
- **参与者**：仅通过分享链接进入（如 `https://example.com/l/abc123`），无需登录。

## 流程一：创建者 · 从零到首次发布

1. 访问首页 → 点击「登录/注册」→ 完成 Clerk 登录/注册。
2. 进入 Dashboard（我的活动列表）。
3. 点击「新建活动」→ 系统创建默认活动（`status: draft`）并跳转到**创作工作台**。
4. **创作工作台（单屏闭环）**：
   - **左栏**：配置表单（标题、slug、奖品、开场文案、兑奖联系人等）。每次修改**自动保存**到草稿，无需手动点"保存"。
   - **右栏**：实时预览参与者视角，创建者所改即所见。
   - 确认满意后，点击预览区上方的**「发布」**按钮 → 活动状态变为 `published`，展开分享面板（复制链接 + 二维码）。
5. 发布后仍可回到工作台修改，修改**即时生效**（已发布状态下的编辑为热更新）。

**体验要点**：
- 新建即进入创作工作台，配置 → 预览 → 发布在**同一页面**完成，零跳转。
- 草稿态下 `/l/[slug]` 返回 404 或"活动即将上线"占位页，避免参与者看到半成品。
- "发布"是一个**显式的心理确认动作**——创建者确认的是"右边预览的效果我满意了"，而非盲目保存。

## 流程二：参与者 · 打开链接到完成抽奖

1. 打开分享链接 `/l/[slug]`（任意设备、无登录）。
2. **开场阶段**（可选）：展示头像 + 三段打字文案，营造氛围。
3. **过渡**：简短下落 emoji 动画。
4. **转盘阶段**：看到活动标题、转盘、奖品列表；点击「开始抽奖」→ 转盘旋转 → 停在随机奖品。
5. **中奖弹窗**：展示中奖内容与兑奖联系人；点击「太棒了」或遮罩关闭；后台写入一条中奖记录。
6. 可再次点击「开始抽奖」继续玩（若业务允许）。

**体验要点**：首屏加载快；转盘与按钮反馈明确；弹窗可关闭且不阻塞；移动端单手可操作。

## 流程三：创建者 · 查看与管理记录

1. 在「我的活动」点击某活动的「记录」→ 进入该活动的抽奖记录页。
2. 查看统计（总次数、各奖品命中次数/占比）、表格（时间、奖品、IP/位置、详情）。
3. 使用排序（最新/最早）、搜索（奖品文案、IP、位置）筛选。
4. 分页浏览；可随时返回「编辑」或「抽奖页」链接。

**体验要点**：表格信息密度适中；搜索与排序即时生效；移动端可横向滚动或卡片化展示。

## 流程四：异常与边界

- **链接无效**：`/l/[slug]` 无对应活动 → 404，提示「活动不存在」并引导回首页。
- **未登录访问后台**：重定向到登录页，登录后回到目标页或 Dashboard。
- **自动保存失败**：工作台左栏顶部短暂展示错误文案（如「slug 已被占用」），不中断操作。
- **发布校验未通过**：点击发布时若缺少必填项（如无奖品），高亮对应字段并在发布按钮下方提示原因，不关闭面板。
- **访问未发布活动**：`/l/[slug]` 若对应活动为 `draft`，展示友好占位页"活动即将上线，敬请期待"，不暴露 404。
- **抽奖页加载失败**：Script 或数据异常时，展示友好错误与重试入口。
