# 03 信息架构与路由设计

## 路由树（建议）

```
/                     # 落地页（未登录）；已登录 → 重定向 /dashboard
/sign-in              # 登录（Clerk）
/sign-up              # 注册（Clerk）

/dashboard            # 我的活动列表（需登录）
/lottery/new          # 新建活动（需登录，服务端创建后重定向编辑）
/lottery/[id]/edit    # 编辑活动（需登录，本人）
/lottery/[id]/logs    # 某活动的抽奖记录（需登录，本人）

/l/[slug]             # 公开抽奖页（无需登录，按 slug 查活动）
```

## 导航结构

- **未登录**：仅落地页 + 登录/注册；落地页 CTA 明显。
- **已登录（Dashboard 壳）**：
  - **侧栏**：品牌名（链回 Dashboard）、「我的活动」、「新建活动」；底部「账号」+ UserButton。
  - **主内容区**：当前路由对应页面（列表 / 编辑 / 记录）；面包屑或返回链在页面内。
- **公开抽奖页**：无侧栏，全屏沉浸；仅抽奖流程与关闭/再玩。

## 权限与可见性

| 路由 | 谁可访问 | 说明 |
|------|----------|------|
| `/`, `/sign-in`, `/sign-up` | 所有人 | 未登录可看落地与登录页 |
| `/dashboard`, `/lottery/*` | 已登录且本人 | 活动列表/编辑/记录均按 `clerk_user_id` 校验 |
| `/l/[slug]` | 所有人 | 仅校验 slug 存在，不校验登录 |

## 页面层级与返回

- 列表 → 编辑 / 记录：从列表卡片进入；编辑页与记录页均提供「返回列表」「打开抽奖页」「查看记录」等交叉入口。
- 新建 → 编辑：服务端创建后直接 `redirect(/lottery/[id]/edit)`，无独立「新建成功」页。
- 404 / 错误页：提供「返回首页」或「重试」，风格与全站一致。
