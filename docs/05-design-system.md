# 05 设计系统

## 设计方向

- **风格**：**弥散光影 + 毛玻璃 (Glassmorphism)**；以低饱和治愈系色彩为基底，前景采用半透明玻璃材质与高光边缘，营造温暖、现代、有质感的视觉体验。
- **基调**：摒弃传统抽奖的高饱和促销风（大红大黄）。整体视觉偏向"治愈系"——柔和、有温度、令人安心，适配生日/节日/礼物场景。
- **一致性**：全站（落地、后台、公开抽奖页）共用一套主色与组件语义；抽奖页在统一色板内通过渐变背景和更大的留白拉开沉浸感，而非另起炉灶。

---

## 配色

### 主色（Primary）

- **主色**：`#ff6f61`（珊瑚/暖橙），用于 CTA、品牌名、关键强调。
  - 在 oklch 中约：`oklch(0.62 0.18 25)`，便于与白/灰搭配。
- **主色悬停**：略亮，如 `#ff856e` 或 `primary / 0.9`。
- **主色深**：用于转盘等需要对比时，如 `#e55b50`。

### 治愈系辅助色板

| 角色 | 色值 | 用途 |
|------|------|------|
| 珊瑚粉 | `#fed6e3` | 抽奖页背景渐变暖端、结果弹窗光晕 |
| 薄荷绿 | `#a8edea` | 抽奖页背景渐变冷端、成功/安心状态 |
| 香槟金 | `#f5e6cc` | 中奖高潮时粒子光点、转盘高光 |
| 淡紫雾 | `#e8dff5` | 可选替代渐变色、生日/梦幻场景 |

辅助色仅用于背景渐变与氛围点缀，不用于文字或交互元素，保证与主色系统不冲突。

### 语义色

- **背景**：浅灰/近白，如 `oklch(0.99 0 0)`；深色模式可用 `oklch(0.145 0 0)`。
- **前景（正文）**：深灰/黑，如 `oklch(0.2 0 0)`；保证与背景对比度 ≥4.5:1。
- **卡片/浮层**：`rgba(255, 255, 255, 0.6)` + `backdrop-filter: blur(16px)`（毛玻璃）；边框用 `rgba(255,255,255,0.3)` 的 1px 高光线，在渐变背景上产生层次。
- **次要/说明**：muted，如 `oklch(0.5 0 0)`。
- **错误/破坏性**：红系，如 `oklch(0.577 0.245 27.325)`，用于错误提示、删除等。
- **成功**：薄荷绿系 `#a8edea` 或绿色，若需单独语义可定义 `--success`。

### 渐变背景系统

抽奖页与落地页使用弥散渐变作为主视觉层：

```css
/* 默认渐变 — 晨曦 */
--gradient-warm: linear-gradient(135deg, #a8edea 0%, #fed6e3 100%);
/* 备用渐变 — 落日 */
--gradient-sunset: linear-gradient(135deg, #f5e6cc 0%, #e8dff5 50%, #a8edea 100%);
```

渐变上叠加慢速呼吸动画（opacity 在 0.8–1 之间缓动），使背景具备生命感而不分散注意力。

### 变量命名建议

- CSS 变量：`--background`、`--foreground`、`--primary`、`--primary-foreground`、`--muted`、`--muted-foreground`、`--border`、`--card`、`--card-glass`（毛玻璃卡片）、`--destructive`、`--ring`、`--gradient-warm`、`--gradient-sunset`、`--glow`（光晕色） 等；Tailwind 通过 `@theme` 或 `theme()` 引用。

## 字体（Typography）

- **西文/数字**：**Geist Sans** 或 Inter，清晰、现代，适合界面与代码。
- **中文**：系统字体栈（如 `PingFang SC`、`Microsoft YaHei`）或选用霞鹜文楷/思源黑体以增强个性；需保证可读性。
- **情感场景（抽奖页开场寄语）**：可引入一款圆润的显示字体（如 `LXGW WenKai` 霞鹜文楷）用于开场文案卡片，与界面字体形成温度反差。
- **字号层级**：
  - 页面标题：1.5rem–2rem，font-weight 700–800。**拉大字重差距**：标题极粗，辅助说明极细，制造视觉呼吸。
  - 卡片/区块标题：1rem–1.25rem，600。
  - 正文：0.875rem–1rem，400。
  - 辅助/说明：0.75rem–0.875rem，300–400，muted 色。
- **行高**：正文约 1.5–1.6；标题可 1.2–1.3。
- **字重**：常规 400，强调 500–600，标题 700–800；在开场卡片等情感区域可用 300 轻体营造优雅感。

## 图标

- **库**：**Lucide React**；统一线性风格、按需引入、与 shadcn 常用组合一致。
- **使用**：导航（如 LayoutList、PlusCircle）、操作（复制、编辑、删除）、状态（成功、错误）等；尺寸与文字或按钮对齐（如 16–24px）。
- **抽奖页**：转盘内奖品图标可由配置的 emoji 或图片承担，与 Lucide 混用时注意风格统一（如统一圆角/线宽）。

---

## 圆角与间距

- **圆角**：全面采用**连续曲率超大圆角 (Squircle / Superellipse)**，对齐 Apple 设计语言，拒绝锐利直角。
  - 基础 `--radius: 0.75rem`（12px）；按钮/输入 `radius-md` (12px)，卡片 `radius-lg` (16px)，弹窗 `radius-xl` (24px)；抽奖 CTA 可用 `rounded-full` 增强点击感。
  - 毛玻璃浮层的圆角统一为 `radius-xl` (24px)，与柔和背景匹配。
  - 实现方式：CSS `border-radius` 已足够；若需更精确的连续曲率，可引入 `figma-squircle` CSS Houdini 库或 SVG `clip-path`。
- **间距**：4/8 的倍数（4、8、12、16、24、32）；区块间 24–32px，表单项间 16px，按钮内 padding 一致。
- **留白哲学**：抽奖页视觉密度远低于管理页。转盘周围至少保留 `48px` 以上留白，让视线聚焦中心。

## 组件规范

- **按钮**：主按钮用 primary 填充，带微妙 `box-shadow` 模拟柔光；次要用 outline 或 ghost；危险操作用 destructive；尺寸 sm/default/lg 与图标尺寸配套。抽奖页 CTA 可加大至 `lg` + `rounded-full`，配合 press 动效。
- **毛玻璃卡片 (Glass Card)**：半透明白底 `rgba(255,255,255,0.55)` + `backdrop-filter: blur(16px) saturate(1.8)` + 1px `rgba(255,255,255,0.3)` 高光边框 + `shadow-lg`（带暖色调）。用于抽奖页开场卡片、结果弹窗、落地页 Hero 区域等情感场景。
- **实体卡片 (Solid Card)**：白/浅底、细边框、统一 padding、shadow-sm；用于 Dashboard、编辑页等管理场景。列表项卡片 hover 可轻微抬升或边框高亮。
- **输入框**：与边框、focus ring 一致；错误态用 destructive 边框与提示文案。
- **表格**：表头与行区分背景；斑马纹可选；移动端可卡片化或横向滚动。
- **弹窗/抽屉**：遮罩半透明黑 `rgba(0,0,0,0.4)` + `backdrop-filter: blur(4px)`；内容区使用毛玻璃卡片样式、超大圆角、最大宽度限制；关闭与主操作按钮位置固定、符合习惯。
- **中奖结果弹窗 (Prize Modal)**：特殊变体——全屏遮罩 + 中心毛玻璃面板，奖品图标放大展示，按钮文案用「收下心意」取代「确定」，底部可选「再来一次」链接。

## 设计 Token 与实现

- 在 `globals.css` 或设计 token 文件中定义 `:root` / `.dark` 的 CSS 变量；Tailwind 通过 `@theme inline` 或 `tailwind.config` 的 `theme.extend` 映射为 `bg-primary`、`text-muted-foreground` 等。
- 组件库（如 shadcn）的默认主题改为使用上述 primary/muted，保证全站一套色板。
- 毛玻璃相关 token 单独归组：`--glass-bg`、`--glass-border`、`--glass-blur`、`--glass-shadow`，方便组件复用。
- 渐变背景 token：`--gradient-warm`、`--gradient-sunset` 在 `:root` 定义，抽奖页与落地页按需引用。

## 视觉分层模型

为保证全站视觉一致而氛围有别，采用三层分离：

| 层级 | 用途 | 视觉特征 |
|------|------|----------|
| **L0 背景层** | 页面底色 | 管理侧：纯色 `--background`；抽奖/落地侧：弥散渐变 + 呼吸动画 |
| **L1 内容层** | 卡片、表单、表格 | 管理侧：实体卡片 (Solid Card)；抽奖/落地侧：毛玻璃卡片 (Glass Card) |
| **L2 浮层** | 弹窗、Toast、结果面板 | 统一毛玻璃 + 更重的模糊 + 遮罩 |

此分层确保设计系统**一套 token 两种氛围**，不需要维护两套独立主题。
