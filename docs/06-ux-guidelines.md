# 06 UX 准则

## 动效总纲

- **核心理念**：动效是情感传递的载体，不是装饰。每一个动画都应回答"它让用户**感受到**了什么？"。
- **物理直觉**：所有运动必须符合真实世界物理感知——有惯性、有重量、有弹性。禁止线性匀速运动。
- **缓动曲线**：统一使用 Spring 物理曲线或 `cubic-bezier(0.34, 1.56, 0.64, 1)`（轻弹性）；入场用 `ease-out`，退场用 `ease-in`，交互反馈用 Spring。

---

## 入场动画编排 (Choreography)

页面不是"一下子出现"的，而是元素按**编排顺序**依次登场，营造"缓缓展开"的温情节奏。

### 抽奖页开场序列

| 序号 | 时机 | 元素 | 动效 | 时长 |
|------|------|------|------|------|
| 1 | 0ms | 渐变背景 | 从 `opacity: 0` 缓慢呼吸至 `1`，背景光晕微幅移动 | 1200ms |
| 2 | 400ms | 创建者头像 | 从 `scale: 0.8, opacity: 0` → `scale: 1, opacity: 1`，Spring 弹性 | 600ms |
| 3 | 700ms | 寄语文字 | 逐字/逐句打字机效果，每字间隔 60–80ms，光标闪烁 | 按字数 |
| 4 | 文字结束后 300ms | "开启好运" 按钮 | 从 `translateY: 20px, opacity: 0` 上浮进入 | 400ms |

### 管理侧入场

- Dashboard 卡片列表：每张卡片 stagger 入场（间隔 50ms），从底部 `translateY: 12px` 上浮 + 淡入，时长 300ms。
- 编辑页：表单区块 stagger 入场，预览手机壳从右侧滑入。

### 落地页入场

- Hero 文字从中心向外扩展 `scale: 0.95 → 1` + 淡入；CTA 按钮延迟 200ms 后 Spring 弹入。

## 微交互 (Micro-interactions)

### 按钮触感

- **Hover**：颜色明度微调 + `translateY: -1px` + 阴影加深，过渡 200ms。
- **Press（Active）**：`scale: 0.96` + 阴影收拢（模拟按压软糖的凹陷感），过渡 100ms。松开时 `scale` 弹回 `1.0`，带 Spring 回弹。
- **触觉反馈 (Haptic)**：移动端在抽奖 CTA 按下时触发 `navigator.vibrate(10)` 轻量震动。

### 复制链接

- 点击后图标从"链接"变为"对勾"，带 `scale` 弹性过渡；Toast 从底部 Spring 弹入，2s 后淡出。

### 表单保存

- 保存按钮点击后短暂显示 Spinner → 成功后变绿色对勾 + "已保存" 文案，1.5s 后恢复原态。

## 转盘物理动效

转盘是整个产品的**高潮体验**，必须有物理质感。

### 启动阶段（蓄力）

- 缓动曲线：`cubic-bezier(0.4, 0.0, 0.2, 1)` — 先慢后快。
- 前 300ms 转盘缓慢启动，给人"沉甸甸"的蓄力感。

### 高速阶段

- 转速达到峰值后匀速持续 1–2s。
- 伴随效果：指针每经过一个奖品格，触发一次微震动 + 清脆的"嗒"音（机械拨盘感）。可选：转盘边缘有微小光粒洒落。

### 减速与停止（阻尼 + 回弹）

- 缓动曲线：`cubic-bezier(0.0, 0.0, 0.2, 1)` + 末尾轻微 **overshoot**（超过目标格约 5° 再弹回），模拟摩擦力与惯性。
- 最终停止时，指针有一次极小的回弹（±2°），暗示机械精密感。

### 停止后

- 短暂静止 500ms（制造悬念），然后触发结果弹窗动画。

## 高潮时刻（中奖结果展示）

摒弃俗套的满屏撒花/彩带雨。追求**有质感的克制**。

### 推荐方案：光晕爆开 + 视差星光

1. **光晕爆开**：从转盘中心向外扩散一圈柔和的主色光晕 (`radial-gradient`)，扩展至屏幕 60%，时长 600ms，然后缓慢消退至 20% 作为结果弹窗的背景光。
2. **视差星光飘落**：10–15 颗香槟金色光点从上方缓慢飘落，前景大且快、背景小且慢、带高斯模糊，营造**景深空间感**。使用 CSS `@keyframes` 或轻量 Canvas，持续 3–4s 后自然消散。
3. **结果面板**：毛玻璃卡片从 `scale: 0.9, opacity: 0` 弹性进入，奖品图标先于文字出现（stagger 100ms）。

### "未中奖" 的温暖处理

- 不要冷冰冰的"很遗憾"。使用温暖插画/emoji（如拥抱的表情）+ 文案如"下次好运一定属于你"或"祝你今天有个好心情"。
- 视觉与中奖弹窗同级别精致，不因结果差异降低体验品质。

## 音效设计

UI 再高级，配上劣质音效也会破功。音效是温情感的最后一环。

| 场景 | 音效描述 | 时长 | 备注 |
|------|----------|------|------|
| 按钮点击 | 柔和水滴/气泡声 | <100ms | 轻量，不抢注意力 |
| 转盘拨动 | 清脆木质/金属嗒声 | <50ms | 每经过一格触发一次 |
| 转盘减速 | 拨动音间隔拉长，自然减速 | 跟随转速 | 物理同步 |
| 中奖结果 | 温暖的钢琴和弦 + 轻铃声 | 1–2s | 情感高峰，不能刺耳 |
| 未中奖 | 轻柔的风铃/木琴下行 | 1s | 温柔，不沮丧 |
| 复制成功 | 短促的"叮" | <200ms | 确认感 |

- **默认静音**：首次访问默认静音，页面右上角提供音效开关（音符图标）。尊重用户偏好。
- **音效格式**：WebM/OGG 优先（体积小），MP3 fallback；单文件 <30KB。

## 通用动效规范

- **时长**：交互反馈 100–200ms；入场/退场 300–600ms；抽奖序列可达 3–5s。
- **禁止**：匀速线性运动、超过 800ms 的非叙事性动画、闪烁/抖动、自动播放的循环动画（背景呼吸除外）。
- **可访问性**：尊重 `prefers-reduced-motion: reduce`，在该媒体查询下将所有非必要动画时长设为 0 或降级为简单淡入。
- **性能**：动效仅使用 `transform` 和 `opacity`（GPU 合成层），禁止 animate `width`/`height`/`top`/`left`。星光粒子使用 Canvas 或 CSS `will-change: transform`。

## 无障碍（A11y）

- **焦点**：所有可交互元素可键盘聚焦；焦点环清晰（如 `ring-2 ring-ring ring-offset-2`），不依赖仅颜色区分。
- **对比度**：正文与背景 ≥4.5:1，大字 ≥3:1；主色按钮上文字与背景满足对比度。
- **语义**：按钮用 `<button>`，链接用 `<a>`；表单有 `<label>` 或 `aria-label`；弹窗设 `role="dialog"`、`aria-modal="true"`、`aria-labelledby`。
- **屏幕阅读器**：图标按钮提供 `aria-label`（如「复制链接」「关闭」）；抽奖结果在弹窗内可读，顺序合理。

## 响应式

- **断点**：与 Tailwind 默认一致（sm 640、md 768、lg 1024、xl 1280）；侧栏在移动端收起到 offcanvas/抽屉，主内容全宽。
- **触控**：按钮与链接最小约 44×44px 可点区域；列表卡片整块可点或主要操作足够大。
- **抽奖页**：转盘与按钮在小屏居中、宽度 max 90vw；弹窗 padding 适配小屏，避免内容被裁切。
- **表格**：在窄屏可横向滚动或改为卡片列表，保证「时间、奖品、位置」等信息可见。

## 加载与错误

- **首屏**：关键内容优先；Lucky Canvas 等脚本延后或懒加载，避免阻塞。
- **加载态**：列表/记录页用 Skeleton 或「加载中…」文案；新建活动跳转编辑前若有延迟可考虑极短 loading（可选）。
- **错误态**：接口失败展示明确文案（如「加载失败，请重试」）+ 重试按钮；表单校验错误在字段旁或顶部汇总，用 destructive 色。
- **404/500**：统一错误页布局，主色标题 + 简短说明 +「返回首页」或「重试」，风格与全站一致。

## 一致性

- **术语**：全站统一「活动」「抽奖」「奖品」「记录」「兑奖」等用词；按钮文案统一（如「保存」「取消」「复制链接」）。
- **操作位置**：主操作（新建、保存、开始抽奖）位置稳定；危险操作（删除）二次确认或放在次要位置。
- **空状态**：无活动时引导「新建活动」；无记录时说明「暂无抽奖记录」，不空白一片。
