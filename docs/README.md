## 使用方式

- **产品 / 业务**：优先阅读 01、02、03。
- **前端 / 设计**：优先阅读 03、05、06、07。
- **全栈 / 架构**：通读 04，并结合 03 做接口与数据建模。

## 原则

- **用户第一**：流程以「创建者」与「参与者」两类角色为中心。
- **简单可扩展**：先满足 MVP，结构上预留扩展（如多活动类型、数据导出）。
- **视觉与交互统一**：一套设计语言（弥散光影 + 毛玻璃）覆盖落地、后台、公开抽奖页，通过视觉分层实现"一套 token 两种氛围"。
