# 07 页面规格

## 落地页 `/`

- **职责**：介绍产品、引导登录/注册；已登录则重定向 `/dashboard`。
- **布局**：全屏单屏，弥散渐变背景 (`--gradient-warm`) + 慢速呼吸动画。中央一块毛玻璃卡片 (Glass Card) 承载所有内容。
- **内容层次**（由上至下，居中排列）：
  1. 品牌 Logo 或文字标识（轻量，不喧宾夺主）。
  2. 主标题（大字重 700–800，主色 `--primary`）。
  3. 一句副标题（轻字重 300–400，muted 色）。
  4. 主 CTA 按钮「开始创建」（`rounded-full`，主色填充，Spring 弹入）。
  5. 底部可选：极简特性说明（3 个图标 + 短文案，水平排列或竖向）。
- **入场动效**：背景先亮 → 毛玻璃卡片从 `scale: 0.95, opacity: 0` 淡入 → 标题与按钮 stagger 入场。见 06 入场编排。
- **要点**：首屏即见 CTA；视觉重心在中央卡片，四周大量留白；移动端卡片宽度 `max-w-sm`。

## 登录/注册 `/sign-in`, `/sign-up`

- **职责**：Clerk 托管；仅做路由与可能的容器样式。
- **要点**：保持 Clerk 组件居中、与全站主色或背景协调；登录后跳转目标页或 Dashboard。

## 我的活动 `/dashboard`

- **职责**：展示当前用户的活动列表；入口为「新建活动」、每项可「复制链接」「抽奖页」「编辑」「记录」。
- **布局**：标题 + 主按钮右对齐；列表为**大卡片网格**（非密集表格），每卡：标题、slug/日期、**状态徽章**（`草稿` 灰色圆角标签 / `已发布` 绿色圆角标签）、参与人次徽章、操作按钮组。卡片用 Solid Card 样式，hover 轻微抬升 (`translateY: -2px`) + 阴影加深。
- **卡片操作差异**：
  - **草稿态**：主操作为「继续编辑」，不显示「复制链接」（链接不可用）。
  - **已发布态**：主操作为「复制链接」，次要操作为「编辑」「记录」。
- **空状态**：不留白一片。展示温暖插画（如礼物盒 / 星星）+ 文案"还没有活动，创建第一个吧" + 虚线边框大卡片内嵌 `+` 图标按钮，整体视觉等同一张普通卡片的位置，降低心理负担。
- **入场**：卡片列表 stagger 上浮淡入，间隔 50ms。
- **要点**：信息层级清晰；复制后 Toast 从底部 Spring 弹入。

---

## 新建活动 `/lottery/new`

- **职责**：服务端创建默认活动（`status: draft`）并重定向至创作工作台 `/lottery/[id]/edit`；无独立 UI 或仅极短 loading。
- **要点**：创建失败时重定向回 Dashboard 或展示错误，不白屏。

---

## 创作工作台 `/lottery/[id]/edit`

这是创建者的**核心页面**。配置、预览、发布在同一屏幕完成，零跳转。心智模型从"填表后台"转变为"创作工具"。

### PC 端布局（左右分栏）

```
┌─────────────────────────────────────────────────────────┐
│  ← 返回列表    活动标题（可编辑）         查看记录      │  ← 顶部工具栏
├───────────────────────────┬─────────────────────────────┤
│                           │  ┌─ 状态徽章 ─┐  ┌─发布─┐  │
│   ┌─ 基础信息 ────────┐  │  │ 草稿 / 已发布│  │ 发布 │  │  ← 发布按钮在预览区上方
│   │ 标题 / Slug       │  │  └────────────┘  └──────┘  │
│   └───────────────────┘  │  ┌─────────────────────┐    │
│                           │  │   ╭───────────────╮ │    │
│   ┌─ 开场与展示 ──────┐  │  │   │  手机壳预览   │ │    │
│   │ 头像 / 寄语文案   │  │  │   │               │ │    │
│   │ Emoji 列表        │  │  │   │  (实时渲染     │ │    │
│   └───────────────────┘  │  │   │   参与者视角)  │ │    │
│                           │  │   │               │ │    │
│   ┌─ 奖品与兑奖 ──────┐  │  │   ╰───────────────╯ │    │
│   │ 奖品列表 (可拖拽) │  │  └─────────────────────┘    │
│   │ 兑奖联系人        │  │                              │
│   └───────────────────┘  │  复制链接 · 打开抽奖页       │  ← 发布后出现分享入口
│                           │                              │
│   左栏 ~55%               │   右栏 ~45% (sticky)         │
└───────────────────────────┴─────────────────────────────┘
```

- **左栏 (~55%)**：配置表单，分三个 Solid Card 区块（基础信息、开场与展示、奖品与兑奖），stagger 入场。
  - **自动保存**：每次字段变更 debounce ~800ms 后自动 PATCH 到服务端。左栏顶部显示微妙的保存状态指示器（"已保存" / "保存中..." / "保存失败，点击重试"），不需要手动保存按钮。
  - slug 冲突等错误在字段旁即时展示。

- **右栏 (~45%, Sticky)**：实时预览 + 发布控制。
  - **顶部**：活动状态徽章（`草稿` 灰色 / `已发布` 绿色）+「发布」主按钮（草稿态）或「取消发布」次要按钮（已发布态）。
  - **中部**：**手机壳实时预览**。带圆角手机边框的容器（iPhone 15 比例 `390×844` 缩放至合适尺寸），内部用同一套抽奖页组件以受控 props 实时渲染参与者视角（开场卡片 → 转盘）。创建者每改一个字段，预览即时响应。预览区可切换查看"开场阶段"与"转盘阶段"。
  - **底部**（发布后可见）：「复制链接」按钮 + 二维码缩略图 +「打开抽奖页」链接。

- **发布流程**：
  1. 创建者点击「发布」→ 前端校验必填项（标题、至少一个奖品）→ 校验不通过则高亮左栏对应字段。
  2. 校验通过 → `PATCH status: published` → 按钮变为绿色对勾 + "已发布" → 底部分享区域 Spring 弹入（复制链接 + 二维码）。
  3. 这是一个**仪式感时刻**：发布成功后可选轻量 confetti 或光晕扩散动效，给创建者成就感。

### 移动端布局

- 单栏。配置表单在上，底部固定一个浮动操作栏：左侧「预览」按钮（打开全屏 Sheet 展示手机预览），右侧「发布」按钮。
- 预览 Sheet 内可左右滑动切换"开场"与"转盘"阶段。

### 技术实现要点

- 预览区**不用 iframe**，而是将抽奖页组件抽离为受控组件（接收 `config` props），在工作台内直接渲染。这样避免 iframe 通信延迟，实现真正的逐字实时更新。
- 预览容器用 CSS `transform: scale()` + `transform-origin: top center` 缩放到合适尺寸，内部组件按真实手机宽度渲染，保证所见即所得。

## 抽奖记录 `/lottery/[id]/logs`

- **职责**：展示某活动的所有中奖记录；支持排序（最新/最早）、搜索、分页；顶部可展示总次数与各奖品统计。
- **布局**：
  - 标题 + 操作按钮（编辑、抽奖页、返回列表）。
  - **统计摘要区**：横向排列的小卡片（总次数、各奖品命中数/占比），数字用大字体 + 主色强调。
  - **工具栏**：搜索框 + 排序下拉，与表格紧邻。
  - **记录列表**：PC 端为表格（时间、奖品图标+文案、位置/IP、详情展开）；**移动端自动切换为时间轴卡片布局**——每条记录一张小卡片，左侧时间线竖线 + 圆点，右侧奖品 + 时间，类似朋友圈消息流，信息密度适中且易读。
  - 分页。
- **空状态**：无记录时展示插画 + "还没有人来抽奖，快去分享链接吧" + 复制链接按钮。
- **要点**：加载/错误态统一；Skeleton 占位与最终内容形状一致。

## 公开抽奖页 `/l/[slug]`

- **职责**：按 slug 展示活动；这是整个产品的**情感巅峰页面**，参与者的全部体验浓缩于此。
- **结构**：全屏沉浸式，**无导航栏、无侧栏、无多余 UI**。弥散渐变背景 + 仅右上角一个静音/音效切换图标。

### 阶段一：开场 (The Hook)

- **布局**：全屏居中，一张毛玻璃明信片卡片 (Glass Card)。
- **内容**（由上至下）：
  1. 创建者头像（圆形，`64–80px`，带轻微 `box-shadow` 光晕）。
  2. 寄语文字（打字机逐字出现，使用情感字体如霞鹜文楷）。
  3. "开启好运" 按钮（`rounded-full`，主色，延迟入场）。
- **视觉**：背景渐变缓慢呼吸；卡片周围大量留白；整体感受如收到一封信。
- **动效**：见 06 开场序列。

### 阶段二：过渡

- 点击按钮后，明信片卡片 `scale: 0.95, opacity: 0` 淡出 → 短暂 emoji 下落过渡（300–500ms）→ 转盘区域从底部 Spring 弹入。

### 阶段三：转盘 (The Draw)

- **布局**：屏幕中央转盘（占视口宽度 `min(80vw, 360px)`），下方"开始抽奖"CTA 大按钮，转盘下方可选：小字奖品列表（折叠/展开）。
- **视觉**：转盘周围至少 `48px` 留白；背景渐变继续但亮度微降，聚焦转盘。
- **动效**：见 06 转盘物理动效（蓄力 → 高速 → 阻尼回弹）。

### 阶段四：结果 (The Reveal)

- **中奖**：光晕爆开 → 视差星光飘落 → 毛玻璃结果面板 Spring 弹入。面板内：奖品图标（放大 `64–96px`）→ 奖品名称 → 兑奖联系人 → 按钮"收下心意"。见 06 高潮时刻。
- **未中奖**：同等精致的毛玻璃面板，温暖 emoji 插画 + "祝你今天有个好心情" 文案 + "再来一次" 按钮。不因结果差异降低体验品质。
- 面板可通过遮罩点击或按钮关闭，回到转盘阶段，可再次抽奖。

### OG 分享图

- 链接在微信/社交平台解析时展示的分享卡片是参与者的**第一印象**。必须在服务端动态生成 OG Image（Canvas / Satori），包含：创建者头像、活动标题、精美渐变背景。这是拉升打开率的关键。

- **要点**：首屏加载快（Lucky Canvas 脚本延迟加载，关键 CSS 内联）；全程无登录无多余步骤；移动端单手可操作。

## 错误页 `error.tsx`, `not-found.tsx`

- **职责**：展示运行时错误或 404；提供重试或返回首页。
- **布局**：居中、渐变背景（与落地页一致），毛玻璃卡片内承载错误信息。
- **内容**：温暖插画/emoji（如迷路的星星）+ 主色标题 + 简短说明 +「返回首页」主按钮。
- **基调**：即使是错误页，也保持产品的温情调性，不冷冰冰。
